================================================================================
                    JUSTIFICATION ARCHITECTURALE (PFE)
================================================================================
  L'API OpenAI (GPT-4o-mini) n'est qu'une pièce du puzzle. Le véritable moteur 
  d'intelligence artificielle est un Pipeline NLP complet :
  
  1. L'Ingestion : Infrastructure Node.js asynchrone pour la réception et le 
     parsing des fichiers binaires (PDF) sans bloquer le serveur principal.
  2. Prompt Engineering (Moteur NLP) : Création d'instructions système complexes
     agissant comme extracteur d'entités nommées (NER). L'IA est bridée pour 
     créer de la donnée JSON structurée exploitable à partir de contenu brut.
  3. L'Automatisation (Logique Métier) : Du dépôt du fichier à l'enregistrement
     MongoDB, le flux est 100% automatisé, sans intervention humaine.
  4. Préparation au Matching : En isolant (normalisation) les compétences des 
     expériences de manière atomique, on pose le socle technique indispensable
     afin que nos algorithmes puissent réaliser le Matching Sémantique (Module 2).

================================================================================
                     DÉFINITION DES ACTEURS ET RÔLES (CONCEPTION UML)
================================================================================

1. LE RECRUTEUR (Acteur Principal)
   - Gestion des offres : Créer, modifier et publier des descriptions de postes (Job Descriptions).
   - Pilotage de l'IA : Lancer l'analyse automatique des CV téléchargés pour extraire les données.
   - Évaluation & Matching : Consulter les scores de compatibilité générés par l'IA pour identifier les meilleurs candidats.
   - Suivi des entretiens : Planifier les rendez-vous et utiliser le Kit d'entretien IA (questions suggérées et points de vigilance).
   - Prise de décision : Changer le statut des candidatures (Admis, Refusé, En attente).

2. LE CANDIDAT
   - Dépôt de candidature : Télécharger son CV (PDF) sur la plateforme.
   - Gestion du profil : Consulter l'état d'avancement de sa candidature (via le tableau de bord candidat).
   - Interaction : Recevoir les notifications concernant les invitations aux entretiens ou les retours sur son dossier.

3. L'ADMINISTRATEUR
   - Gestion des utilisateurs : Contrôler les accès, créer ou suspendre des comptes.
   - Maintenance du système : Surveiller les statistiques globales d'utilisation.
   - Configuration : Gérer les paramètres techniques (clés API OpenAI, limites).

4. LE SERVICE IA (Acteur Système)
   - Extraction Sémantique (Parsing) : Transformation du texte brut PDF en objets JSON structurés.
   - Calcul de Score (Matching) : Comparaison sémantique candidat ↔ offre.
   - Génération de contenu : Production des kits d'entretien personnalisés (Prompt Engineering).

================================================================================
                     SÉCURITÉ ET AUTHENTIFICATION (PRODUCTION)
================================================================================

[✅] SÉCURITÉ RÉELLE ACTIVÉE
    - `User.js` : Hachage automatique des mots de passe via `bcryptjs` (salt 10).
    - `authController.js` : Vérification stricte des identifiants (comparaison des hashs).
    - `authMiddleware.js` : Vérification réelle du JWT et récupération de l'utilisateur en base.
    - Conséquence : Seuls les utilisateurs enregistrés avec le bon mot de passe peuvent accéder à la plateforme.

================================================================================
                     GÉNÉRATION DES MODÈLES MONGOOSE (ARCHITECTURE DE DONNÉES)
================================================================================

[✅] CRÉATION DES 5 MODÈLES PRINCIPAUX (Dossier : /BACKEND/models/)
    - Modèle User.js : Champs (nom, email, password), avec un rôle strictement limité à ['candidate', 'recruiter', 'admin'].
    - Modèle Job.js : Offres d'emploi avec (titre, description, compétences, lieu, salaire) et référence 1:N vers le User (Recruteur).
    - Modèle Candidate.js : Lié au User. Intègre les structures complexes (tableaux d'objets sans _id) pour 'experiences', 'formations', et 'skills' extraites par parsing IA.
    - Modèle Application.js : Entité de liaison entre Candidate et Job, avec un 'status' (Pending, Reviewed, Interviewed, Rejected) et le 'scoreMatching' (Number).
    - Modèle KitEntretien.js : Relié via relation 1:1 à Interview. Stocke le 'resumeIA', 'questionsTechniques', et 'pointsVigilance'.
    => *Toutes les entités utilisent mongoose.Schema avec validations (trim, required, unique), timestamps: true et format d'export CommonJS.*
